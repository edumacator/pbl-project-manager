# Architecture Rules

1. Preserve portability of the data layer.
2. Respect and extend the repository pattern already in use.
3. Separate business logic from transport and UI concerns.
4. Favor small, composable modules over monolithic feature blocks.
5. Avoid tight coupling between feature verticals.
6. Keep migrations reversible and easy to reason about.
7. Preserve auditability for meaningful workflow actions.
8. Keep the UI responsive and use progressive disclosure.
9. Extend existing models before creating parallel systems.
10. Before introducing a new pattern, verify reuse is insufficient and explain why.
