# Project Brain

This directory serves as the persistent operating context for AI coding agents and human contributors. It preserves product intent, architectural standards, and pedagogical requirements to ensure consistency across development sessions and prevent architecture drift.

## Recommended Read Order

To understand the project context, contributors should read these files in the following order:

1. [codex-operating-rules.md](codex-operating-rules.md) - How to work with this project.
2. [product-vision.md](product-vision.md) - What this product is (and isn't).
3. [design-principles.md](design-principles.md) - UI/UX philosophy and student-centered design.
4. [architecture-rules.md](architecture-rules.md) - Technical constraints and patterns.
5. [workflow-rules.md](workflow-rules.md) - How we use AI tools and manage tasks.
6. [feature-guardrails.md](feature-guardrails.md) - Preventing scope creep and functional drift.
7. [pedagogy-rules.md](pedagogy-rules.md) - Educational theory driving the features.
8. [decision-log.md](decision-log.md) - History of major technical and product choices.
9. [task-template.md](task-template.md) - Standard format for defining new work.
